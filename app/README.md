# IIG Reporter

Управление рассылкой отчётов Яндекс.Директ → Telegram: обнаружение чатов, привязка к клиентам,
цели/атрибуция, предпросмотр и отправка отчётов, история. **Десктоп-окно и веб-версия — из одного кода.**

PowerShell-скрипт (`weekly_report.ps1`) не сломан и продолжает работать; переезд можно делать постепенно.

## Вариант А — без Python (рекомендуется): один .exe
1. Скачай **`IIGReporter.exe`** из раздела **Releases** репозитория (сборка «Последняя сборка — IIGReporter.exe»)
   или из вкладки **Actions** → последний прогон → артефакт `IIGReporter`.
   *(exe собирается автоматически на GitHub Actions — Python на твоём ПК не нужен.)*
2. Положи рядом с exe файл **`secrets.json`** (токены Telegram и Яндекса; шаблон — `secrets.example.json`).
   По желанию рядом же — `config.json` (тексты/атрибуция) и `app_config.json` (админы).
3. Запусти:
   - двойной клик по `IIGReporter.exe` → **десктоп-окно**;
   - `IIGReporter.exe web` → **веб-версия** (откроется в браузере, `http://127.0.0.1:8077`);
   - `IIGReporter.exe weekly` → разовая рассылка (это вешается в «Планировщик задач» на Пн 09:00).

   Десктоп-окну нужен системный **Edge WebView2 Runtime** (на Win10/11 обычно есть). Если окна нет —
   используй веб-версию, она работает в любом браузере.

## Вариант Б — с Python (для разработки)
```
cd app
pip install -r requirements.txt
run.bat            # десктоп-окно   (или: python -m iigbot)
run.bat web        # веб-версия
run.bat weekly     # рассылка
```
Токены берутся из общего `secrets.json` рядом с `weekly_report.ps1`.

## Экраны
- **Обзор** — счётчики и блок «Требует внимания».
- **Клиенты** — список из Директа; клик по строке → карточка (цели, атрибуция, чаты, тест-отчёт).
- **Чаты** — автоматически обнаруженные чаты; привязка к клиенту.
- **Привязки** — система предлагает совпадение чат↔клиент, подтверждаешь в один клик.
- **Отчёты** — предпросмотр сообщения (формат 1:1 с текущим) и история; «Отправить тест», «Запустить рассылку».
- **Настройки** — статусы токенов, админы, тексты, атрибуция, расписание.
- **＋ Подключить клиента** — выдаёт deep-link, по которому бот добавляется в чат и привязывается сам.

## Первый проход
1. Запусти приложение. 2. **Клиенты → «Подтянуть из Директа»** (или «Импорт config.json»).
3. Добавь бота в чат клиента → он появится в **Чаты**. 4. **Привязки** — подтверди.
5. **Отчёты → Предпросмотр** → «Отправить тест». 6. Когда ок — «Запустить рассылку сейчас».

## Автозапуск рассылки
`IIGReporter.exe weekly` (или `run.bat weekly`) — в «Планировщике задач» Windows на нужный день/время.

## Запуск .exe на Windows и частые проблемы
**Просто запусти `IIGReporter.exe`** (рядом — `secrets.json`). Программа сама поднимает фоновый
слушатель Telegram отдельным процессом (`IIGReporter.exe bot`, вывод в `listener.log`) и открывает окно.
Отдельный процесс используется потому, что в onefile-GUI-сборке (`console=False`) поток-слушатель
внутри окна не запускается; одиночный лок не даёт развестись дубликатам слушателя.
Лаунчеры `IIG_Запуск.bat`/`_iig_listener.bat` остаются как запасной вариант, но обычно не нужны.

Чек-лист, если «не работает»:
- **`secrets.json` должен лежать РЯДОМ с `IIGReporter.exe`** и называться именно так (мн. число).
  Туда же кладутся `iigbot.sqlite3` (база) и `app_config.json` (создаётся сам).
- **Статус «Нет связи» в окне = программа не достучалась до Telegram** (это вызов `getMe`).
  Если Telegram заблокирован в стране — **VPN должен покрывать ВЕСЬ трафик системы (full-tunnel),
  а не только браузер/приложение Telegram (split-tunnel)**, иначе программа идёт мимо VPN и режется.
  Проверка: внешний IP системы должен быть «не-российским». После включения VPN жми «⟳ Обновить».
- **Не запускай несколько копий** — иначе бот отвечает дублями (конфликт `getUpdates`).
- Яндексу VPN не нужен — он работает напрямую.

## Безопасность
- `secrets.json`, `app_config.json`, база `iigbot.sqlite3` в git не попадают (`.gitignore`).
- Впиши свой Telegram ID в Настройки → «Администраторы» — тогда `/bind` в Telegram сможешь только ты
  (защита от ошибочной привязки чужого чата).

## Как это собирается в .exe
`.github/workflows/build-exe.yml` на каждый push в ветку запускает Windows-раннер, ставит зависимости и
PyInstaller, собирает `app/iigreporter.spec` в один файл `IIGReporter.exe` и публикует его в артефакты и в
релиз `latest`. Локально (если есть Python): `cd app && pip install pyinstaller && pyinstaller iigreporter.spec`.

## Структура
```
app/
  main.py                точка входа для exe
  iigbot/
    cli.py / __main__.py подкоманды (desktop/web/weekly/sync/import/bot)
    desktop.py           окно pywebview + фоновый слушатель
    web.py               веб-сервер (тот же ui.html + /api)
    ui.html              интерфейс (работает и в окне, и в браузере)
    api.py               backend (база/Директ/Telegram)
    report.py            движок отчётов (порт weekly_report.ps1)
    bot.py / listener.py слушатель Telegram (обнаружение чатов)
    storage.py telegram_api.py yandex.py settings.py
    sync_clients.py import_config.py run_weekly.py
  iigreporter.spec  requirements.txt  app_config.example.json  run.bat
```
