# Еженедельный отчёт Яндекс.Директ → Telegram

Скрипт берёт статистику за прошлую неделю (Пн–Вс) по списку клиентов
из Reports API Яндекс.Директа и шлёт каждому в свой Telegram-чат через бота.
Работает на твоей машине, по расписанию Windows — Claude и браузер не нужны.

## Файлы
- `weekly_report.py` — основной скрипт
- `config.json` — токены и список клиентов (заполнить)
- `get_chat_id.py` — помощник, чтобы узнать chat_id чата
- `run_weekly_report.bat` — запуск для планировщика Windows

## 0. Куда положить
Создай папку, например `C:\Users\trutn\ad-reports`, и положи туда все 4 файла.

## 1. Python + библиотека
Если Python ещё не стоит — поставь с python.org (галочка «Add to PATH»).
Затем один раз установи зависимость, в командной строке:
```
pip install requests
```

## 2. Токен Яндекс.Директа (OAuth)
1. Зайди на https://oauth.yandex.ru/client/new под аккаунтом агентства.
2. Создай приложение: дай любое имя, в правах (scope) выбери **Яндекс.Директ (direct:api)**,
   платформа — «Веб-сервисы», Redirect URI — `https://oauth.yandex.ru/verification_code`.
3. Скопируй **ClientID** приложения и открой в браузере:
   `https://oauth.yandex.ru/authorize?response_type=token&client_id=ВСТАВЬ_CLIENTID`
4. Подтверди — в адресной строке вернётся `access_token=...`. Это и есть токен.
   Скопируй `secrets.example.json` в `secrets.json` и вставь токен в `yandex_oauth_token`.
   (Файл `secrets.json` в git не попадает — он в `.gitignore`.)

> Примечание: у аккаунта должен быть доступ к API Директа. У агентских (Директ Про)
> он обычно есть. Если API недоступен — это включается на стороне Яндекса.

## 3. Telegram-бот
1. В Telegram напиши **@BotFather** → `/newbot` → задай имя и юзернейм.
2. BotFather пришлёт **токен бота** — вставь его в `secrets.json` → `telegram_bot_token`.

## 4. chat_id чата клиента
1. Создай (или возьми существующий) чат с названием клиента, например «simplefoods».
2. Добавь туда своего бота. Чтобы бот видел сообщения группы, у BotFather
   выполни `/setprivacy` → выбери бота → **Disable** (или сделай бота админом чата).
3. Напиши в чате любое сообщение с упоминанием бота, например `@твой_бот привет`.
4. Запусти помощник:
   ```
   python get_chat_id.py
   ```
   Он покажет `chat_id` (у групп он отрицательный, вроде `-1001234567890`).
   Вставь его в `config.json` → clients → `chat_id`.

(По желанию: впиши свой личный chat_id в `owner_chat_id` — туда придут уведомления об ошибках.)

## 5. Тестовый запуск
```
python weekly_report.py
```
Отчёт должен прийти в чат. Лог пишется в `report.log`.

## 6. Расписание (Пн 09:00)
Командная строка от имени администратора, путь подставь свой:
```
schtasks /Create /SC WEEKLY /D MON /ST 09:00 /TN "Weekly Direct Report" /TR "C:\Users\trutn\ad-reports\run_weekly_report.bat"
```
Или через «Планировщик заданий» → Создать простую задачу → Еженедельно → Пн, 09:00 →
Запустить программу → указать `run_weekly_report.bat`.

> Задача сработает, если компьютер включён в это время. В планировщике можно
> включить «Выполнить при следующем включении, если запуск был пропущен».

## Добавить ещё клиентов
Просто допиши объекты в массив `clients` в `config.json`
(`name`, `login` — логин клиента в Директе, `chat_id` — его чат). Скрипт менять не нужно.
