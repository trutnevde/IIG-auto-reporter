# HANDOFF — IIG Reporter (передача между сессиями/устройствами)

> Этот файл — «точка входа» для продолжения работы на другом устройстве или в новой сессии.
> Память Claude локальна (на конкретной машине) и НЕ переезжает — поэтому всё важное здесь.

## Как продолжить на другом устройстве
1. Склонировать/обновить репозиторий: `git clone https://github.com/trutnevde/IIG-auto-reporter` (или `git pull`).
2. Открыть сессию Claude Code в папке репозитория.
3. Сказать ассистенту: **«Прочитай HANDOFF.md и продолжай».**
4. Рядом с exe должен лежать `secrets.json` (токены) — он НЕ в репозитории. Без него можно править код, но не тестировать вживую.

Последний коммит на момент написания: **1c9057c** (ветка `main`).

## Что это за проект
IIG Reporter — десктоп/веб-приложение (PyInstaller onefile `IIGReporter.exe`), которое рассылает
еженедельные отчёты Яндекс.Директ → Telegram по клиентам агентства. Собирается на GitHub Actions,
exe лежит в релизе `latest`. Уже в боевом использовании (десятки клиентов).

## Текущее состояние (работает)
- Один exe = интерфейс + фоновый слушатель Telegram (отдельным процессом). `secrets.json` рядом с exe,
  токены можно вводить прямо в Настройках.
- Клиенты подтягиваются из Директа кнопкой; привязки чат↔клиент; еженедельная рассылка;
  удаление чатов-«висяков» (🗑 во вкладке «Чаты»); вкладка «Гайд».
- **Конструктор отчётов** (Отчёты → Конструктор): разрезы account/campaign/adgroup/ad/keyword/searchquery,
  9 моделей атрибуции (LSC/LC/FC/LYDC/LSCD/LCD/FCD/LYDCD/AUTO), свободные даты, копировать/отправить в чат.

## Критичные факты окружения
- **Telegram (`api.telegram.org`) заблокирован у пользователя (РФ, DPI)** → нужен VPN в режиме «весь трафик»
  (full-tunnel). Яндекс работает напрямую. Симптом блокировки: статус «Нет связи», ошибки отправки.
- **База `iigbot.sqlite3` лежит рядом с exe и СВОЯ для каждой папки/пользователя Windows.** Привязки/цели
  не переезжают между аккаунтами Windows сами — копировать этот файл.
- **Один бот = один слушатель** (getUpdates отдаёт обновления только одному). Нельзя держать две запущенные
  копии одного бота одновременно — будут дубли/конфликты.

## Токены
- `secrets.json`: `telegram_bot_token` + `yandex_oauth_token` (последний — ТОЛЬКО Директ).
- Токен Яндекса **не имеет доступа к Метрике** (Management API → 403). Для целей из Метрики нужен новый
  OAuth-токен с правом на Метрику (тот же тип токена, просто приложение с включённой Метрикой).

## Ключевые файлы
- `app/iigbot/report_custom.py` — движок конструктора (LEVELS, ATTRIBUTION_MODELS, `build()`, `to_text()`).
- `app/iigbot/report.py` — недельный движок + `fetch_report(report_type=...)`.
- `app/iigbot/api.py` — backend `Api` (report_options/report_query/report_send, delete_chat, save_secrets, sync_clients, preview, run_weekly, history, settings…).
- `app/iigbot/ui.html` — интерфейс (одна страница; функции renderBuilder/renderChats/renderReports/…). Светлая/тёмная тема.
- `app/iigbot/listener.py` + `bot.py` — слушатель отдельным процессом + одиночный лок (127.0.0.1:49517).
- `app/iigbot/storage.py` — SQLite (chats, clients, bindings, send_log, kv).
- `app/iigbot/settings.py` — пути/секреты/CA-bundle/шаблон secrets.json.
- `.github/workflows/build-exe.yml` — сборка exe → релиз `latest`.

## Как собирать/тестировать
- Правка кода → `git push` в `main` → GitHub Actions собирает exe (~1–2 мин) → скачать из релиза `latest`.
- exe в песочнице запустить нельзя; валидировать так: `python -m py_compile`, импорт модулей, и **живые вызовы
  Reports API** с токеном из `secrets.json` (как в этой сессии) ДО пуша.
- Перед правками **всегда `git fetch`/`pull`** — над репозиторием параллельно работает и другая сессия.

## Полезные разрезы/типы Reports API (проверено)
- Типы отчётов: ACCOUNT/CAMPAIGN/ADGROUP/AD/CRITERIA(фразы)/SEARCH_QUERY _PERFORMANCE_REPORT — все работают.
- Поля-метрики: Impressions, Clicks, Cost, Ctr, AvgCpc, Conversions (работает без Goals!).
- Измерения: CampaignName, AdGroupName, AdId, Criterion, Query, Device, Gender, Age, LocationOfPresenceName, AdNetworkType.
- SEARCH_QUERY и гео дают ОЧЕНЬ много строк (десятки тысяч) — нужен топ-N/пагинация.
